# قالب Flutter التجريبي

تطبيق Flutter لإدارة المنتجات والملف الشخصي، مبني على Clean Architecture ويتصل بواجهة Laravel REST API.

---

## المتطلبات

قبل تشغيل التطبيق، تأكد من توفر:

- Flutter SDK `^3.9.2` أو أحدث
- Dart SDK متوافق مع المشروع
- محرر أكواد (VS Code / Android Studio)
- جهاز محاكي أو جهاز فعلي (Android / iOS)
- خادم Laravel API يعمل ومتاح

---

## إعداد المشروع

### 1. تثبيت الحزم

```bash
flutter pub get
```

### 2. توليد الملفات المطلوبة (عند الحاجة)

```bash
dart run build_runner build --delete-conflicting-outputs
```

### 3. ضبط عنوان الـ API

افتح الملف:

`lib/core/config/api_config_repository.dart`

وعدّل قيمة `_defaultBaseUrl` لتطابق عنوان خادم الـ API لديك:

```dart
static const String _defaultBaseUrl =
    'http://localhost/laravel-api-dashboard/public/api/v1';
```

> **ملاحظة:** إذا كنت تختبر على جهاز محاكي Android، استخدم `10.0.2.2` بدلاً من `localhost` للوصول إلى الخادم على جهازك.

## نظرة عامة على التطبيق

بعد فتح التطبيق، ستجد الشاشات والميزات التالية:

| القسم | الوصف |
|-------|--------|
| تسجيل الدخول | الدخول بحساب موجود |
| إنشاء حساب | تسجيل مستخدم جديد |
| المنتجات | عرض وإضافة وتعديل وحذف المنتجات |
| الملف الشخصي | عرض بياناتك وتعديل الاسم وتسجيل الخروج |

---

## خطوات تشغيل واستخدام التطبيق

### 1. إنشاء حساب جديد

1. افتح التطبيق — ستظهر شاشة **تسجيل الدخول**.
2. اضغط على **"ليس لديك حساب؟ سجل الآن"**.
3. أدخل البيانات التالية:
   - **الاسم**
   - **البريد الإلكتروني**
   - **كلمة المرور**
   - **تأكيد كلمة المرور**
4. اضغط **إنشاء حساب**.
5. عند نجاح التسجيل، سيتم توجيهك تلقائياً إلى الشاشة الرئيسية.

---

### 2. تسجيل الدخول

1. من شاشة **تسجيل الدخول**، أدخل:
   - **البريد الإلكتروني**
   - **كلمة المرور**
2. اضغط **تسجيل الدخول**.
3. عند نجاح الدخول، ستنتقل إلى الشاشة الرئيسية.

> إذا كان لديك حساب بالفعل، يمكنك الانتقال من شاشة التسجيل إلى تسجيل الدخول عبر **"لديك حساب بالفعل؟ سجل الدخول"**.

---

### 3. الشاشة الرئيسية

بعد تسجيل الدخول، تظهر الشاشة الرئيسية بشريط تنقل سفلي يحتوي على:

- **المنتجات** — إدارة قائمة المنتجات
- **الملف الشخصي** — عرض وتعديل بياناتك

---

### 4. إدارة المنتجات

#### عرض المنتجات

1. من الشريط السفلي، اختر **المنتجات**.
2. ستظهر قائمة المنتجات مع:
   - الاسم
   - السعر
   - الكمية
   - الصورة (إن وُجدت)
3. اسحب للأسفل لتحديث القائمة (**Pull to Refresh**).
4. عند وجود منتجات كثيرة، يتم تحميل المزيد تلقائياً عند الوصول لنهاية القائمة.

#### البحث عن منتج

1. استخدم حقل **بحث في المنتجات** أعلى الشاشة.
2. اكتب اسم المنتج — سيتم البحث تلقائياً بعد كتابة النص.
3. لمسح البحث، اضغط أيقونة **X** داخل حقل البحث.

#### إضافة منتج جديد

1. اضغط زر **+** (Floating Action Button) في شاشة المنتجات.
2. املأ البيانات:
   - **صورة المنتج** (اختياري — من المعرض أو الكاميرا)
   - **الاسم** (مطلوب)
   - **الوصف** (اختياري)
   - **السعر** (مطلوب)
   - **الكمية** (مطلوب)
3. اضغط **حفظ**.
4. عند النجاح، ستظهر رسالة تأكيد ويعود التطبيق لقائمة المنتجات.

#### تعديل منتج

1. من قائمة المنتجات، اضغط أيقونة **تعديل** بجانب المنتج.
2. عدّل البيانات المطلوبة.
3. اضغط **حفظ**.

#### حذف منتج

1. من قائمة المنتجات، اضغط أيقونة **حذف** بجانب المنتج.
2. أكّد الحذف في نافذة التأكيد.
3. عند النجاح، ستظهر رسالة **تم حذف المنتج بنجاح**.

---

### 5. الملف الشخصي

1. من الشريط السفلي، اختر **الملف الشخصي**.
2. ستظهر بياناتك:
   - **الاسم**
   - **البريد الإلكتروني**

#### تعديل الاسم

1. اضغط أيقونة **تعديل** بجانب الاسم.
2. عدّل الاسم في الحقل الظاهر.
3. اضغط **حفظ** للتأكيد، أو **إلغاء** للتراجع.
4. عند النجاح، ستظهر رسالة **تم تحديث الملف الشخصي بنجاح**.

#### تحديث البيانات

- اسحب الشاشة للأسفل لتحديث بيانات الملف الشخصي.

#### تسجيل الخروج

1. اضغط زر **تسجيل الخروج** في أسفل الشاشة.
2. سيتم إنهاء الجلسة وإعادتك إلى شاشة تسجيل الدخول.

## واجهات الـ API المستخدمة

| العملية | الطريقة | المسار |
|---------|---------|--------|
| تسجيل | POST | `/api/v1/register` |
| دخول | POST | `/api/v1/login` |
| خروج | POST | `/api/v1/logout` |
| الملف الشخصي | GET | `/api/v1/profile` |
| تحديث الملف | PUT | `/api/v1/profile` |
| قائمة المنتجات | GET | `/api/v1/products` |
| إضافة منتج | POST | `/api/v1/products` |
| تعديل منتج | PUT | `/api/v1/products/{id}` |
| حذف منتج | DELETE | `/api/v1/products/{id}` |
